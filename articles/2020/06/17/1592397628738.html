webpack4构建多页应用

2020-06-19   |   0 评论   |   0 浏览

使用方式

git clone http://git.ym/g-web/webpack-mpa-template.git

一、配置分离：webpack 配置文件说明

位于根目录的 config 文件夹下：

webpack.base.conf.js：主要配置打包所需的 entry 入口、module 的 rules、externals 配置 jQuery 等常用开发库、公共的 plugins（静态资源输出、消除冗余的 css/js 代码、自动生成 HTML 模板等）。

webpack.dev.conf.js：主要配置 mode 环境变量、devServer 快速开发、HMR 模块热替换等。

webpack.prod.conf.js：主要配置 mode 环境变量、正式环境打包配置优化（html/js/css 压缩与合并等）。

webpack.rules.conf.js：配置常规 loader。

二、多页面入口配置（核心）

webpack 打包入口支持单入口和多入口，由于我们的项目页面数量是未知的，将所有页面都枚举在配置里或者手动配置都是不合理的，我们需要定义一个 getEntry()自动化 方法来遍历指定文件夹(本文多页目录设置在 src/pages 下)获取所有的入口 js 文件路径，给 entry 传入对象即可。

//当前位置：config\webpack.base.conf.js
//动态添加入口
function getEntry(PAGES_DIR) {
	var entry = {};
	//读取src目录所有page入口
	glob.sync(PAGES_DIR + '**/*.js').forEach(function (name) {
		var start = name.indexOf('pages/') + 4;
		var end = name.length - 3;
		var n = name.slice(start, end);
		n = n.split('/')[1];
		entry[n] = name;
	})
	return entry;
}
let entrys = getEntry('./src/pages/')
module.exports = {
	entry: entrys
}

三、打包输出配置

无论是单入口还是多入口，都只能指定一个输出配置。以下是项目的 output 配置。其中，对于需要配置 cdn 访问的静态资源，可以修改 publicPath 路径为 cdn 路径，这些打包后的资源对外的的根目录就是 publicPath。

output: {
        path: path.resolve(__dirname, '../dist'),
        // 打包多出口文件
        // 生成 a.bundle.[hash].js  b.bundle.[hash].js
        filename: './js/[name].[chunkhash:8].js',
        publicPath: './'
},

四、项目目录结构

我们的多页面开发目录安排在了 src/pages 下，每一个新页面目录的命名格式，统一是以 index 来命名（好处就是利于生成相对应的 HTML 文件，统一命名管理）。而 src/js、src/css、src/images 目录用于存放公共文件，src/assets 用于存放不用 webpack 打包的静态资源。

五、wbepack 主要配置如下：

5.1 配置开发服务器

//依赖安装：npm install webpack-dev-server --save-dev
//当前位置：config\webpack.base.conf.js
devServer: {
        contentBase: path.join(__dirname, "../dist"),
        publicPath:'/',
        host: "127.0.0.1",
        port: "8089",
        overlay:{// 浏览器页面上显示错误
            errors:true,
            warnings:true,
        },
        open: true, // 开启浏览器
        openPage:"demo1.html",
        // stats: "errors-only", //stats: "errors-only"表示只打印错误：
        hot: true, // 开启热更新
        // hotOnly:true //hot 和 hotOnly 的区别是在某些模块不支持热更新的情况下，前者会自动刷新页面，后者不会刷新页面，而是在控制台输出热更新失败
}

在 package.json 添加 webpack 启动命令如下：

"scripts": {
  "test": "echo \"Error: no test specified\" && exit 1",
  "dev": "cross-env NODE_ENV=development webpack-dev-server  --config config/webpack.dev.conf.js",
  "build": "cross-env NODE_ENV=production webpack --config config/webpack.prod.conf.js",
  "server": "live-server ./ --port=8090",
  "c": "node config/create.js",
  "start": "npm run dev"
}

5.3 从 js 中分离 CSS

webpack 4.0 以前，是通过 extract-text-webpack-plugin 插件把 CSS 样式从 js 文件中提取到单独的 CSS 文件中。extract-text-webpack-plugin 插件的优缺点：

webpack 4.0 以后，官方推荐使用 mini-css-extract-plugin 插件来打包 CSS 文件：

//依赖安装：npm install  mini-css-extract-plugin --save-dev
//当前位置：config\webpack.prod.conf.js// 分离css插件参数为提取出去的路径
new miniCssExtractPlugin({
filename: 'css/[name].[contenthash:8].min.css',
}),

5.4 处理 css3 属性前缀和移动端适配，消除冗余的 CSS 代码并压缩 CSS

使用移动端适配插件 lib-flexible，如需兼容 pc，此处需要修改本地依赖配置。/node_modules/lib-flexible/flexible.js 的 refreshRem 方法 width = 540* dpr 的限制改为 width = width* dpr;（本地配置每次 npm i 后都要改一次，也可以通过 cdn 的引入方式一次修改）

//依赖安装： npm install postcss-loader postcss-px2rem --save-dev
//属性自动加前缀
//在根目录新建postcss.config.js
module.exports = {
    plugins: [
        require('autoprefixer'),//自动添加css前缀
        require('postcss-px2rem')({
            remUnit: 75,
            remPrecision: 2 // rem的小数点后位数
          })
    ]
};
//loader里加入postcss
{
        test: /\.(css|scss|sass)$/,
        use: [
            devMode ? 'style-loader' : {
                loader: MiniCssExtractPlugin.loader,
                options: {
                    // you can specify a publicPath here
                    // by default it use publicPath in webpackOptions.output
                    publicPath: '../'
                }
            },
            'css-loader',
            'postcss-loader',
            'sass-loader',
        ]
    }

修改 package.json 加入 CSS 要兼容浏览器版本的代码，autoprefixer 不会对 border-radius 自动添加前缀，浏览器基本上已经支持 border-radius 属性，除了 ie6-ie8、Opera10.1 这几个版本不支持，其他浏览器已经都支持 border-radius 属性。

"browserslist": [
    "defaults",
    "not ie < 11",
    "last 2 versions",
    "> 1%",
    "iOS 7",
    "last 3 iOS versions"
]

消除冗余 CSS 代码 使用 glob 模块去匹配文件所以还要安装 glob 模块

//依赖安装：npm install purifycss-webpack --save-dev
//当前位置：config\webpack.base.conf.js
new purifyCssWebpack({
    paths: glob.sync(path.join(__dirname, "../src/pages/*/*.html"))
}),

压缩 CSS

//依赖安装：npm install optimize-css-assets-webpack-plugin --save-dev

5.5 清空打包目录

在构建前清除打包目录中的内容

//依赖安装：npm install clean-webpack-plugin --save-dev
//当前位置：config\webpack.prod.conf.js
//删除dist目录
new cleanWebpackPlugin(['dist'], {
    root: path.resolve(__dirname, '../'), //根目录
    // verbose Write logs to console.
    verbose: true, //开启在控制台输出信息
    // dry Use boolean "true" to test/emulate delete. (will not remove files).
    // Default: false - remove files
    dry: false,
}),

5.6 压缩 js

上线压缩 去除 console 等信息，webpack4.x 之后去除了 webpack.optimize.UglifyJsPlugin

//依赖安装：npm install uglifyjs-webpack-plugin --save-dev
//当前位置：config\webpack.prod.conf.js
new UglifyJSPlugin({
    uglifyOptions: {
        compress: {
            //warnings: false,  ////使用 terser-webpack-plugin 代替，实现生产去除 console.log
            drop_debugger: false,
            drop_console: true
        }
    }
}),

5.7 提取 js 公共模块

webpack4 里面移除了 commonChunksPulgin 插件，放在了 config.optimization 里面进行配置来提取 js， vendor 名字可以自定义修改。

//当前位置：config\webpack.base.conf.js
optimization: {
    splitChunks: {
        cacheGroups: {
            vendor: {
                // test: /\.js$/,
                test: /[\\/]node_modules[\\/]/,
                chunks: "initial", //表示显示块的范围，有三个可选值：initial(初始块)、async(按需加载块)、all(全部块)，默认为all;
                name: "vendor", //拆分出来块的名字(Chunk Names)，默认由块名和hash值自动生成；
                enforce: true,
            }
        }
    }
}

5.8 复制静态资源

处理不需要使用 webpack 统一打包处理或 webpack 不支持的文件，统一放入 assets 目录管理，将 src 目录下的 assets 文件夹复制到 dist 目录下

//依赖安装：npm install copy-webpack-plugin --save-dev
//当前位置：config\webpack.base.conf.js
new copyWebpackPlugin([{
    from: path.resolve(__dirname, "../src/assets"),
    to: './assets',
    ignore: ['.*']
}]),

5.9 自动生成 HTML 文件（核心）

通过 js 入口文件的多个 key 名，遍历配置相对应个数的 htmlWebpackPlugin 生成 HTML。

//当前位置：config\webpack.base.conf.js
 
// 获取html-webpack-plugin参数的方法
let getHtmlConfig = function (name, chunks) {
    return {
        template: `./src/pages/${name}/index.html`,
        filename: `${name}.html`,
        // favicon: './favicon.ico',
        // title: title,
        inject: true,
        hash: true, //开启hash  ?[hash]
        chunks: chunks,
        minify: process.env.NODE_ENV === "development" ? false : {
            removeComments: true, //移除HTML中的注释
            collapseWhitespace: true, //折叠空白区域 也就是压缩代码
            removeAttributeQuotes: true, //去除属性引用
        },
    };
};
 
var htmlArray = [];
Object.keys(entrys).forEach(function (element) {
    htmlArray.push({
        _html: element,
        title: '',
        chunks: [element]
    })
})
 
//自动生成html模板
htmlArray.forEach((element) => {
    module.exports.plugins.push(new htmlWebpackPlugin(getHtmlConfig(element._html, element.chunks)));
})

5.10 性能优化：可视化分析模块

通过使用 webpack-bundle-analyzer 可以看到项目各模块的大小，可以按需优化。


//依赖安装：npm install webpack-bundle-analyzer --save-dev
//当前位置:webpack.prod.conf.js
if(process.env.npm_config_report){//打包后模块大小分析//npm run build --report
    const BundleAnalyzerPlugin = require('webpack-bundle-analyzer').BundleAnalyzerPlugin;
    webpackConfigProd.plugins.push(new BundleAnalyzerPlugin())
}
